# TabHub - 本地标签管理插件 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
TabHub - 本地标签管理插件

### 1.2 项目目标
开发一个开源的Chrome浏览器标签管理插件，用于替换默认的新标签页，并提供分类管理标签的功能。

### 1.3 技术栈
- Chrome Extension (Manifest V3)
- HTML + CSS + JavaScript
- 纯本地开发，不依赖后端服务器

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 新标签页替换
- **功能描述**：安装插件后，所有新打开的标签页默认加载插件的首页（`index.html`）。
- **实现方式**：使用Chrome Extension的`chrome_url_overrides`功能覆盖默认的新标签页。
- **边界条件**：仅替换新标签页，不影响现有标签页。

#### 2.1.2 标签分类管理
- **功能描述**：页面按分类（Category）展示用户添加的网页链接。
- **默认数据**：插件安装时，内置一份默认的`tabs.json`，包含常用网站（如GitHub、Google、掘金等）。
- **添加标签**：用户可手动添加新链接，填写：名称、URL、图标、选择分类。
- **编辑/删除**：对单个标签进行编辑、删除操作。
- **分类管理**：新增、重命名、删除分类。
- **批量管理**：
  - 支持导出当前所有标签为`tabs.json`文件。
  - 支持导入外部`tabs.json`文件，完全覆盖现有数据。

### 2.2 数据存储
- **存储方式**：纯本地JSON文件，使用`chrome.storage.local`存储。
- **数据结构**：标准JSON格式，包含分类和标签信息。
- **数据安全**：所有数据仅存储在本地，不向任何服务器发送。

## 3. 非功能需求

### 3.1 性能要求
- 页面加载速度快，响应迅速。
- 数据操作（添加、编辑、删除）响应时间不超过1秒。

### 3.2 兼容性
- 兼容最新版Chrome/Edge浏览器。
- 支持Manifest V3标准。

### 3.3 可用性
- 界面简洁、直观，易于操作。
- 提供清晰的用户反馈。

## 4. 数据结构

### 4.1 标签数据结构
```json
{
  "categories": [
    {
      "id": "1",
      "name": "工作",
      "tabs": [
        {
          "id": "1-1",
          "name": "GitHub",
          "url": "https://github.com",
          "icon": "https://github.com/favicon.ico"
        }
      ]
    }
  ]
}
```

### 4.2 字段说明
- **categories**：分类数组
  - **id**：分类唯一标识
  - **name**：分类名称
  - **tabs**：标签数组
    - **id**：标签唯一标识
    - **name**：标签名称
    - **url**：标签URL
    - **icon**：标签图标URL

## 5. API依赖

### 5.1 Chrome Extension API
- **chrome.storage.local**：用于本地数据存储。
- **chrome_url_overrides**：用于覆盖默认的新标签页。

### 5.2 浏览器API
- **FileReader**：用于读取导入的JSON文件。
- **Blob**：用于生成导出的JSON文件。
- **URL.createObjectURL**：用于创建文件下载链接。

## 6. 功能边界

### 6.1 已明确的功能
